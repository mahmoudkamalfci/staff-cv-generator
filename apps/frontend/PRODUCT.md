# Product

## Register

product

## Users
- **Internal Staff**: Manage their basic profile (name, job title, photo, experience summary), list their skills, and describe their roles and contributions in specific project experiences.
- **Project Managers**: Manage projects (details, client, location, duration, technologies used), link staff members to projects with roles and contributions, and generate CVs using predefined templates.

## Product Purpose
To capture staff skills and project participation in a structured way and automatically compile them into professional, tender-ready CVs. The system eliminates manual CV formatting, ensuring consistency and speed when GISCON submits bids for software development proposals.

## Brand Personality
- **Softer, Modern Directory**: Warm but clean layout, generous whitespace, soft visual structure that feels approachable yet highly professional.
- **Production-Grade Polish**: Real working code, committed visual details, robust interactive elements, and complete states (loading, errors, success). No draft-like shortcuts.

## Anti-references
- **Draft/Prototype aesthetics**: Unstyled HTML controls, generic mock cards, harsh side-stripe borders, or basic unpolished lists.
- **Cluttered utility-overkill**: Rigid, overly dense tables with no rhythm, harsh high-contrast black borders everywhere, or visual noise.

## Design Principles
1. **Tender-Ready A4 Print Standard**: The ultimate value of the app is the exported CV. The CV preview and output must perfectly adapt to standard A4 printing constraints (proper margins, page breaks, no orphans) and render in strict, professional black-and-white print styling.
2. **Professional Editorial Spacing**: Use clean typography scales and structured vertical rhythm to frame information clearly without defaulting to generic cards or nested containers.
3. **Comprehensive Interface Integrity**: Every view must handle all edge cases (empty states for new staff/projects, form validation feedback, clean error recovery) to feel like a production system.

## Accessibility & Inclusion
- High contrast (>= 4.5:1) for digital reading, and clean contrast for black-and-white print output.
- Keyboard navigable controls for form entries and selection dialogs.
- Reduced motion support (`@media (prefers-reduced-motion: reduce)`) for all transitional animations.
